## 5. AUTHENTICATION FLOWS

### Registration Flow
```
1. User submits email, name, password
2. Check if email exists (409 if exists)
3. Hash password (bcrypt, 12 rounds)
4. Create User record (role: USER)
5. Create Session record
6. Generate access token (JWT)
7. Generate refresh token (crypto)
8. Set refresh token cookie (HttpOnly, Secure)
9. Log auth event (LOGIN_SUCCESS)
10. Return user + access token + sessionId
```

### Login Flow
```
1. User submits email, password
2. Find user by email
3. Check user status (must be ACTIVE)
4. Verify password (bcrypt.compare)
5. Create Session record
6. Generate access token (JWT)
7. Generate refresh token (crypto)
8. Set refresh token cookie (HttpOnly, Secure)
9. Update user.lastLoginAt
10. Log auth event (LOGIN_SUCCESS)
11. Return user + access token + sessionId
```

### Refresh Flow
```
1. Extract refresh token from cookie or body
2. Hash refresh token (SHA-256)
3. Find session by token hash
4. Validate session (ACTIVE, not expired, user ACTIVE)
5. Generate new refresh token (rotation)
6. Update session (new token hash, lastSeenAt, IP, UA)
7. Generate new access token (JWT)
8. Set new refresh token cookie
9. Log auth event (TOKEN_REFRESH)
10. Return user + access token + sessionId
```

### Logout Flow
```
1. Extract sessionId from request
2. Update session status to REVOKED
3. Set revokedAt timestamp
4. Log auth event (LOGOUT)
5. Clear refresh token cookie
6. Return loggedOut: true
```

### Session Revoke Flow
```
1. Extract sessionId from request
2. Verify session belongs to user
3. Update session status to REVOKED
4. Set revokedAt timestamp
5. Log auth event (SESSION_REVOKED)
6. Return revoked: true
```
