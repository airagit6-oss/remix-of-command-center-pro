## Enforcement Engine

### Typography Lint Rules — ENFORCED

**Rule 1: No Inline Typography Styling**
- **Violation:** `style={{ fontSize: '16px', fontWeight: 'bold' }}`
- **Correct:** `className="body-md font-semibold"`
- **Enforcement:** Linter will flag inline font-size, font-weight, line-height, font-family

**Rule 2: No Arbitrary Font Sizes**
- **Violation:** `fontSize: '18px'`
- **Correct:** Use typography tokens or utility classes
- **Enforcement:** Linter will flag arbitrary pixel/rem/em values for font-size

**Rule 3: No Arbitrary Font Weights**
- **Violation:** `fontWeight: 550`
- **Correct:** Use font-weight tokens or utility classes (400, 500, 600, 700)
- **Enforcement:** Linter will flag arbitrary font-weight values

**Rule 4: No Arbitrary Line Heights**
- **Violation:** `lineHeight: 1.4`
- **Correct:** Use line-height tokens or utility classes (1.15, 1.3, 1.5, 1.65)
- **Enforcement:** Linter will flag arbitrary line-height values

**Rule 5: No Hardcoded Typography Values**
- **Violation:** `const styles = { fontSize: '16px' }`
- **Correct:** Use CSS variables or utility classes
- **Enforcement:** Linter will flag hardcoded typography values in style objects

**Rule 6: No Skipped Heading Levels**
- **Violation:** Using display-3xl followed by heading-md
- **Correct:** Follow heading hierarchy (display → heading-lg → heading-md)
- **Enforcement:** Linter will warn about skipped heading levels

**Rule 7: No Inconsistent Hierarchy**
- **Violation:** Using body-lg for metadata
- **Correct:** Use meta-lg for metadata
- **Enforcement:** Linter will warn about inconsistent hierarchy usage

## Design System Hardening

### Block Inline Typography Styling — BLOCKED

**What's Blocked:**
- Inline `style={{ fontSize: '16px' }}`
- Inline `style={{ fontWeight: 'bold' }}`
- Inline `style={{ lineHeight: 1.5 }}`
- Inline `style={{ fontFamily: 'Arial' }}`

**What's Allowed:**
- Typography utility classes
- CSS variables from tokens.css
- Tailwind typography utilities (if configured with tokens)

### Block Arbitrary Text Utilities — BLOCKED

**What's Blocked:**
- Arbitrary Tailwind classes like `text-[17px]`
- Arbitrary font weights like `font-[550]`
- Arbitrary line heights like `leading-[1.4]`
- Custom utility classes not in governance

**What's Allowed:**
- Locked typography utility classes
- Tailwind utilities mapped to tokens
- CSS variables from tokens.css

### Block Random Font Scaling — BLOCKED

**What's Blocked:**
- Random font sizes not in scale
- Inconsistent font scaling across components
- Skipping font scale levels without justification

**What's Allowed:**
- Locked font scale (text, heading, display, body, metadata)
- Consistent font scaling across components
- Proper scale progression

### Block Hierarchy Inconsistency — BLOCKED

**What's Blocked:**
- Using body text for headings
- Using heading text for body
- Using body text for metadata
- Using metadata for body text
- Skipping hierarchy levels without justification

**What's Allowed:**
- Proper hierarchy (display → heading → body → metadata)
- Consistent hierarchy across components
- Justified hierarchy exceptions

### Block Unmanaged Typography Overrides — BLOCKED

**What's Blocked:**
- Global CSS overrides for typography
- Component-level typography overrides without governance
- Theme-level typography changes without approval
- Direct manipulation of DOM typography styles

**What's Allowed:**
- Token-based typography changes
- Governance-approved typography updates
- Theme-level typography via tokens
- Component-level typography via utility classes
